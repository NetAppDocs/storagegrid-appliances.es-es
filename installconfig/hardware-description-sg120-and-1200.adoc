= Dispositivos StorageGRID SG120 y SG1200
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
El dispositivo de servicios StorageGRID SG120 y el dispositivo de servicios SG1200 pueden funcionar como Nodo de puerta de enlace y como Nodo de administración para proporcionar servicios de equilibrio de carga de alta disponibilidad en un sistema StorageGRID. Ambos dispositivos pueden funcionar como Nodos de puerta de enlace y Nodos de administración (primarios o no primarios) al mismo tiempo.



== Funciones de los dispositivos

Los dispositivos SG120 y SG1200 ofrecen las siguientes características:

* Funciones del nodo de puerta de enlace o del nodo de administración para un sistema StorageGRID.
* El instalador de dispositivos StorageGRID para simplificar la puesta en marcha y la configuración de nodos.
* Cuando se implementa, puede acceder al software StorageGRID desde un nodo de administración existente o desde el software descargado en una unidad local. Para simplificar aún más el proceso de implementación, se incluye una versión reciente del software en el dispositivo durante la fabricación.
* Un controlador de administración en placa base (BMC) para supervisar y diagnosticar parte del hardware del dispositivo.
* La capacidad de conectarse a las tres redes StorageGRID, incluidas la red de grid, la red de administración y la red de cliente:
+
** El SG120 admite hasta cuatro conexiones de 10 o 25 GbE a la Grid Network y a la Client Network.
** El SG1200 admite hasta cuatro conexiones de 10, 25, 40 o 100 GbE a la Grid Network y a la Client Network (o hasta cuatro conexiones de 200 GbE con una NIC de 200 GbE opcional).

== Aplicaciones SG120 y SG1200

Puedes configurar los dispositivos de servicios StorageGRID de las siguientes formas para proporcionar servicios de gateway, así como redundancia de algunos servicios de administración de grid.

* Agregue a una cuadrícula nueva o existente como nodo de puerta de enlace
* Añada a un grid nuevo como nodo de administrador principal o no primario, o a un grid existente como nodo de administrador no primario
* Opere como un nodo de puerta de enlace y un nodo de administración (principal o no primario) al mismo tiempo


El dispositivo facilita el uso de grupos de alta disponibilidad (ha) y el equilibrio de carga inteligente para las conexiones de la ruta de datos S3 o Swift.

Los siguientes ejemplos describen cómo puede maximizar las funcionalidades del dispositivo:

* Usa dos dispositivos SG120 o dos SG1200 para proporcionar servicios de gateway configurándolos como Gateway Nodes.
+

IMPORTANT: Mezclar dispositivos de servicios con diferentes niveles de rendimiento en el mismo sitio, como un SG110 o SG120 con un SG1100 o SG1200, puede provocar resultados impredecibles e incoherentes al utilizar varios nodos en un grupo de alta disponibilidad o al equilibrar la carga de clientes entre varios dispositivos de servicios

* Utiliza dos dispositivos SG120 o dos SG1200 para proporcionar redundancia de algunos servicios de administración de grid. Haz esto configurando cada dispositivo como Admin Nodes.
* Utiliza dos dispositivos SG120 o dos SG1200 para proporcionar servicios de equilibrio de carga y conformación de tráfico de alta disponibilidad a los que se accede a través de una o varias direcciones IP virtuales. Haz esto configurando los dispositivos como cualquier combinación de Admin Nodes o Gateway Nodes y añadiendo ambos nodos al mismo grupo de alta disponibilidad.
+

IMPORTANT: Si utilizas un puerto exclusivo de Admin Node y Gateway Nodes en el mismo grupo de alta disponibilidad, el puerto exclusivo de Admin Node no realizará failover. Consulta las instrucciones de https://docs.netapp.com/us-en/storagegrid/admin/configure-high-availability-group.html["Configurar grupos de alta disponibilidad"^].



Cuando se utilizan con dispositivos de almacenamiento StorageGRID, tanto el dispositivo de servicios SG120 como el SG1200 permiten desplegar redes solo de dispositivos sin depender de hipervisores externos ni de hardware informático.
